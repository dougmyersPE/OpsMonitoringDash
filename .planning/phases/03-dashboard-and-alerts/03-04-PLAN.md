Build the Docker image for the React frontend and update docker-compose + Nginx to serve it.

**Step 1: Create `frontend/Dockerfile`** — multi-stage build: Node.js build stage → nginx:alpine serve stage.

```dockerfile
# Stage 1: Build React SPA
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
# VITE_API_BASE_URL uses relative /api/v1 — Nginx proxies to backend, no absolute URL needed
RUN npm run build

# Stage 2: Serve static files via nginx
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

**Step 2: Create `frontend/nginx.conf`** — SPA fallback so React Router handles all routes client-side:

```nginx
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;

    # Serve static assets directly
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # SPA fallback: all unknown paths return index.html for React Router
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

**Step 3: Update `docker-compose.yml`** — add frontend service after backend.

Add this service to the services block (it does NOT expose its own port — traffic routes through the outer nginx):
```yaml
  frontend:
    build: ./frontend
    depends_on:
      - backend
```

**Step 4: Update `nginx/nginx.conf`** — the outer Nginx that exposes port 80.

This is the major change: add `frontend` upstream, update location blocks to:
1. Proxy `/api/v1/stream` with SSE-safe headers (no buffering)
2. Proxy `/api/*` to backend
3. Proxy `/` to frontend (React SPA)

Replace the entire nginx/nginx.conf with:
```nginx
events {
    worker_connections 1024;
}

http {
    upstream backend {
        server backend:8000;
    }

    upstream frontend {
        server frontend:80;
    }

    server {
        listen 80;

        # SSE endpoint — CRITICAL: disable buffering or events batch silently
        # See RESEARCH.md Pitfall 1: Nginx Buffering Breaks SSE Silently
        location /api/v1/stream {
            proxy_pass http://backend;
            proxy_buffering off;
            proxy_cache off;
            proxy_http_version 1.1;
            proxy_set_header Connection '';
            chunked_transfer_encoding off;
            proxy_read_timeout 86400s;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }

        # All other API routes → FastAPI backend (with normal buffering)
        location /api/ {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        # React SPA — all other requests to frontend container
        # (frontend nginx handles SPA fallback to index.html)
        location / {
            proxy_pass http://frontend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
    }
}
```

**Step 5: Rebuild and start the full stack**
```bash
docker compose build frontend
docker compose up -d
```
