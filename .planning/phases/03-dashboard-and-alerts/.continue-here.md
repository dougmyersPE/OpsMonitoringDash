## Post-phase bug fixes (not in any plan — live debugging session)
All fixes are in `backend/app/workers/poll_prophetx.py`:

1. **Backend container rebuild** — stream, notifications, health/workers endpoints were 404 because backend image was 20hrs stale. Fixed with `docker compose up --build -d backend`.

2. **get_markets 400 Bad Request** — `get_markets_raw()` was called with no `event_id`. ProphetX sandbox requires event_id. Fixed: now fetches events first, extracts event_ids, then calls `get_markets_raw(event_id=eid)` per event and flattens results.

3. **Events response shape** — Response is `{"data": {"sport_events": [...]}}` (two levels deep). Worker only unwrapped one level, getting `{"sport_events": [...]}` treated as a single event. Fixed: unwrap both levels.

4. **Markets event_id injection** — Market objects in `{"data": {"event_id": X, "markets": [...]}}` don't carry event_id on each market. Fixed: inject `eid` into each market dict before flattening.

5. **Sport column always "unknown"** — Worker tried `raw_event.get("sport")` but ProphetX field is `sport_name`. Fixed: added `sport_name` to lookup chain in both INSERT and UPDATE paths.

**Decision made:** Use The Odds API (not API-Sports — free plan blocks current season).

**API key:** `718be09f76addce4b12944f03c9be0d6` (add to .env as `ODDS_API_KEY`)
**Rate limit:** Poll every 10 minutes (not 30s) to conserve free tier credits (500/month).
**Upgrade path:** $59/mo for 100K credits when ready for production polling.

**Sport key mapping** (ProphetX sport_name → Odds API sport_key):
- Basketball → basketball_nba
- Ice Hockey → icehockey_nhl
- MMA → mma_mixed_martial_arts
- Soccer → multiple keys (soccer_epl, soccer_usa_mls, etc.) — needs league-level mapping
- Baseball → baseball_mlb (preseason now, regular season starts March)
- Golf → not available in The Odds API
- Tennis → tennis_atp_dubai (limited)
- American Football → americanfootball_ncaaf

**Response shape from Odds API:**
```json
{
  "id": "...",
  "sport_key": "basketball_nba",
  "sport_title": "NBA",
  "commence_time": "2026-02-26T00:41:00Z",
  "completed": true,
  "home_team": "Memphis Grizzlies",
  "away_team": "Golden State Warriors",
  "scores": [
    {"name": "Memphis Grizzlies", "score": "112"},
    {"name": "Golden State Warriors", "score": "133"}
  ],
  "last_update": "2026-02-26T10:57:25Z"
}
```

**Implementation approach:**
1. Add `ODDS_API_KEY` to `.env` and `config.py`
2. Create `backend/app/clients/odds_api.py` — new client, GET /v4/sports/{sport_key}/scores/
3. Create `backend/app/workers/poll_odds_api.py` — new Celery task
4. Register task in `celery_app.py` with RedBeat schedule: every 10 minutes
5. The worker fetches scores per sport, matches events to DB by home+away team name + date, updates `real_world_status` field on matched events
6. Consider: does the Event model need additional fields for Odds API score data?

**Matching strategy:** fuzzy match on home_team + away_team + date against existing Event rows. ProphetX stores competitor names in `competitors` array — need to check Event model fields.
