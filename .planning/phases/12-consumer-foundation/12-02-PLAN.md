Create `backend/app/workers/opticodds_consumer.py` as a standalone consumer module mirroring the structure of `ws_prophetx.py`. The module has 7 sections:

**Section 1: Imports and module state**
```python
import json
import logging
import random
import signal
import sys
import time
from datetime import datetime, timezone

import httpx
import pika
import redis as _sync_redis
import structlog
from slack_sdk.webhook import WebhookClient

from app.core.config import settings

log = structlog.get_logger()

_message_count = [0]  # mutable counter for D-10 raw logging
UNKNOWN_STATUS_DEDUP_TTL = 300  # 5-minute dedup window for unknown status Slack alerts
```

**Section 2: Status mapping dict (per D-05)**
```python
_OPTICODDS_CANONICAL: dict[str, str] = {
    "not_started":   "not_started",
    "scheduled":     "not_started",
    "delayed":       "not_started",
    "start_delayed": "not_started",
    "postponed":     "not_started",
    "in_progress":   "live",
    "live":          "live",
    "suspended":     "live",
    "interrupted":   "live",
    "finished":      "ended",
    "complete":      "ended",
    "retired":       "ended",
    "walkover":      "ended",
    "cancelled":     "ended",
    "abandoned":     "ended",
}
```

**Section 3: Queue lifecycle helpers**

`_start_queue() -> str`: POST to `settings.OPTICODDS_BASE_URL` with header `X-Api-Key: {settings.OPTICODDS_API_KEY}`, timeout=15. On success, parse `resp.json()["queue_name"]` (field name is MEDIUM confidence per D-10), cache in Redis as `opticodds:queue_name` (no TTL — persists until overwritten), log at INFO `opticodds_queue_started`. On failure (HTTPStatusError or ConnectionError), log at CRITICAL and call `sys.exit(1)` per D-02 (abort on failure).

`_stop_queue() -> None`: Derive stop URL by replacing `/start` with `/stop` in `settings.OPTICODDS_BASE_URL`. POST with same `X-Api-Key` header, timeout=10. Best-effort — catch all exceptions, log at WARNING `opticodds_queue_stop_failed`. This is called during SIGTERM shutdown.

**Section 4: Slack alert helper for unknown statuses (per D-04)**

`_alert_unknown_status(raw_status: str, message_body: str) -> None`:
- If `settings.SLACK_WEBHOOK_URL` is falsy, log WARNING `opticodds_slack_not_configured` and return.
- Create Redis client via `_sync_redis.from_url(settings.REDIS_URL)`.
- Build dedup key: `f"opticodds_unknown_status:{raw_status}"`.
- Attempt `r.set(dedup_key, "1", ex=UNKNOWN_STATUS_DEDUP_TTL, nx=True)`. If returns False (already sent within window), return silently.
- Build message text:
  ```
  :warning: *OpticOdds unknown tennis status*
  *Status:* `{raw_status}`
  *Body (truncated):* `{message_body[:500]}`
  Investigate and add to `_OPTICODDS_CANONICAL` mapping if needed.
  ```
- Send via `WebhookClient(settings.SLACK_WEBHOOK_URL).send(text=text)`.
- Wrap the entire webhook call in try/except, log WARNING `opticodds_slack_alert_failed` on failure (best-effort, never crash the consumer).

This mirrors the `poll_critical_check.py` pattern: direct `slack_sdk.webhook.WebhookClient` with Redis SETNX deduplication. The consumer is a standalone (non-Celery) process, so it cannot use `send_alerts.run.delay()`.

**Section 5: Redis helpers (mirror ws_prophetx.py with rmq/opticodds prefix)**

`_write_heartbeat()`: Set `worker:heartbeat:opticodds_consumer` = `"1"` with ex=90.

`_write_connection_state(state: str)`: Set `opticodds:connection_state` = state with ex=120. Set `opticodds:connection_state_since` = `datetime.now(timezone.utc).isoformat()` with ex=120.

`_write_last_message_at()`: Set `opticodds:last_message_at` = `datetime.now(timezone.utc).isoformat()` with ex=90.

All three functions create Redis client via `_sync_redis.from_url(settings.REDIS_URL)`.

**Section 6: Message callback**

`_on_message(ch, method, properties, body)`: 
- In try block:
  - If `_message_count[0] < 5`: log at DEBUG `opticodds_raw_message` with `body=body.decode()[:1000]`, increment counter (D-10).
  - Parse `data = json.loads(body)`.
  - Extract `raw_status = data.get("status")`.
  - Look up `mapped = _OPTICODDS_CANONICAL.get(raw_status)` if raw_status is truthy.
  - If raw_status is truthy and mapped is None: log at WARNING `opticodds_unknown_status` with `raw_status=raw_status`, then call `_alert_unknown_status(raw_status, body.decode()[:1000])` (D-04: Slack alert for unknown statuses).
  - Call `_write_last_message_at()`.
  - Call `ch.basic_ack(delivery_tag=method.delivery_tag)` (D-08: manual ack).
  - Log at DEBUG `opticodds_message_processed`.
- In except block: log exception `opticodds_message_error`, call `ch.basic_nack(delivery_tag=method.delivery_tag, requeue=True)`.

NOTE: This task does NOT write to the DB — that is Phase 13 scope (TNNS-02 fuzzy matching). The consumer in Phase 12 receives, acks, and logs messages. The `_write_opticodds_status` DB write function will be added in Phase 13.

**Section 7: Main loop**

`run() -> None`:
- Call `queue_name = _start_queue()` (fatal on failure per D-02).
- Initialize `retry_delay = 5`, `max_delay = 60`.
- Infinite while loop:
  - try:
    - Create `pika.ConnectionParameters` with:
      - `host="v3-rmq.opticodds.com"`
      - `port=5672`
      - `virtual_host="api"`
      - `credentials=pika.PlainCredentials(settings.OPTICODDS_RMQ_USERNAME, settings.OPTICODDS_RMQ_PASSWORD)`
      - `heartbeat=30` (D-07)
      - `blocked_connection_timeout=300` (D-07)
    - Create `pika.BlockingConnection(params)`.
    - Get channel, set `basic_qos(prefetch_count=10)`.
    - Call `basic_consume(queue=queue_name, on_message_callback=_on_message, auto_ack=False)` (D-08).
    - Call `_write_connection_state("connected")`.
    - Reset `retry_delay = 5`.
    - Log INFO `opticodds_rmq_connected`.
    - Call `ch.start_consuming()` (blocks forever).
  - except `(pika.exceptions.AMQPConnectionError, pika.exceptions.AMQPChannelError)`:
    - `_write_connection_state("reconnecting")`.
    - Log WARNING `opticodds_rmq_disconnected` with error and retry_in.
    - `time.sleep(retry_delay + random.uniform(0, 1))` (D-09: jitter).
    - `retry_delay = min(retry_delay * 2, max_delay)`.
  - except Exception:
    - Log exception `opticodds_rmq_unexpected_error`.
    - `time.sleep(retry_delay)`.
    - `retry_delay = min(retry_delay * 2, max_delay)`.

**Section 8: Entry point**

```python
if __name__ == "__main__":
    logging.basicConfig(level=logging.INFO, format="%(asctime)s [pika] %(message)s", stream=sys.stdout)
    log.info("opticodds_consumer_starting")

    def _shutdown(sig, frame):
        log.info("opticodds_consumer_shutdown", signal=sig)
        _stop_queue()
        sys.exit(0)

    signal.signal(signal.SIGTERM, _shutdown)
    signal.signal(signal.SIGINT, _shutdown)
    run()
```
