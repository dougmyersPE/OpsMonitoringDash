<completed_work>

- ESPN tennis parsing fixed (commit efe44bb) — ESPN wraps tennis matches in
  `event.groupings[].competitions[]` not `event.competitions[]`. Parser now
  handles both structures. 42/43 tennis events matched immediately.
- Odds API dynamic tennis keys (commit b4ed9cf) — tennis sport keys rotate per
  tournament (e.g., `tennis_atp_dubai` → `tennis_atp_indian_wells`). Now
  auto-discovers active keys from `/v4/sports/` (free endpoint), cached 6h in
  Redis. Also added reversed-name matching for individual sports.
- Odds API key updated — new key with 100k quota (old key was exhausted at 20k).
  Updated in production .env and local .env.
- SDIO task stacking fix (commits 5c996e8, 2792c72) — SDIO poll_sports_data was
  taking 16+ minutes per run (sequential soccer comp fetches). Beat fires every
  30s so multiple instances stacked, consuming all 6 worker slots and starving
  ESPN/Sports API. Added Redis lock so only one SDIO instance runs at a time.
- Heartbeat TTL floor (commit 5c996e8) — all worker heartbeats now have a 600s
  minimum TTL (was `interval * 3` which could be as low as 90s). Prevents
  dashboard from showing "disconnected" during worker contention.
- Source toggles: SDIO disabled, Odds API enabled in production system_config.
</completed_work>

<decisions_made>

- Dynamic Odds API tennis keys over hardcoded: tournament keys change every few
  weeks, hardcoding is always stale. `/v4/sports/` is free (0 quota cost).
- Redis NX lock for SDIO over Celery `acks_late` or `solo` pool: simplest
  solution, lock auto-expires after 20min if task crashes.
- 600s heartbeat TTL floor: generous enough to survive any realistic worker
  starvation without requiring per-worker tuning.
- Disabled SDIO per user request (too slow, flooding workers).
</decisions_made>
