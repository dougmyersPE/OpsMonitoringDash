**docker-compose.yml** — 5 services following Pattern 5 from the research file exactly:
- `postgres`: image postgres:16-alpine, env vars from .env, named volume postgres_data, healthcheck using `pg_isready -d ${POSTGRES_DB} -U ${POSTGRES_USER}` (interval: 5s, timeout: 5s, retries: 10, start_period: 10s)
- `redis`: image redis:7-alpine, command `redis-server --maxmemory 256mb --maxmemory-policy allkeys-lru`, healthcheck using `redis-cli ping` (interval: 5s, timeout: 3s, retries: 10)
- `backend`: build ./backend, command `bash -c "python -m app.seed && alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port 8000"`, env_file: .env, depends_on postgres + redis both with `condition: service_healthy`
- `worker`: build ./backend, command `celery -A app.workers.celery_app worker --loglevel=info -Q default`, env_file: .env, depends_on postgres + redis with `condition: service_healthy`
- `beat`: build ./backend, command `celery -A app.workers.celery_app beat --scheduler redbeat.RedBeatScheduler --loglevel=info`, env_file: .env, depends_on redis with `condition: service_healthy`
- `nginx`: image nginx:alpine, ports 80:80, volume ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro, depends_on: backend

Note: worker and beat services are declared here even though their implementation is in Plan 01-03. They will fail gracefully (import error) until Plan 01-03 creates celery_app.py. This is intentional — declare all services in one place.
